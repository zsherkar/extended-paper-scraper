# Paper Explorer — Design Spec

A web app for browsing and exploring ~60K academic papers across 26 conferences (2023-2026). Core use case: see which papers are cited the most per conference. Hosted as a static site on GitHub Pages.

## Pages

### Home (`/`)
- Grid of conference cards grouped by year (2026, 2025, 2024, 2023)
- Each card shows: venue name, paper count, top 3 cited papers as teaser
- Cards link to the conference page

### Conference (`/conference/:id`)
- Citation leaderboard table: rank, title, authors, track, citation count
- Filters: track (oral/spotlight/poster/main/findings), keyword search within conference
- Sort: by citations (default) or title
- Click paper title to expand: shows abstract + link to PDF
- Papers without citation data show "—" and sort to bottom

### Author (`/author/:name`)
- All papers by an author across conferences, grouped by venue/year
- Summary: total papers, total citations, venues published in

### Trends (`/trends`)
- Keyword frequency across years (bar/line chart, top-20 keywords per year)
- Venue comparison: paper counts per venue over time

## Data Pipeline

### Build script (`scripts/build_data.py`)
Reads JSONL outputs and produces static JSON for the web app:

| Output | Content | Size estimate |
|--------|---------|---------------|
| `web/public/data/manifest.json` | Conference list with metadata (paper count, year, venue, has_citations) | ~2KB |
| `web/public/data/<conference_id>.json` | All papers for one conference | 200KB-1.5MB each |
| `web/public/data/authors.json` | Author -> {conferences, paper_count, total_citations} | ~500KB |
| `web/public/data/trends.json` | Pre-aggregated keyword counts per year, venue paper counts per year | ~10KB |

**Source priority:** `papers_with_citations.jsonl` if it exists, else `papers.jsonl`.

### Client-side loading
- Home: loads only `manifest.json`
- Conference: lazy-loads one `<id>.json` on demand
- Author: loads `authors.json`, then fetches conference files as needed
- Trends: loads `trends.json`

No search index library — `String.includes()` filtering on title/author/keywords is sufficient for per-conference datasets (<6K papers).

## Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Framework | React + TypeScript | User preference |
| Build tool | Vite | Fast dev, simple config |
| Routing | React Router | Client-side SPA routing |
| Charts | Recharts | Lightweight, React-native charting |
| CSS | Custom CSS with CSS variables | Distinctive look, dark mode support |
| Deployment | GitHub Pages via GitHub Action | Free, runs `build_data.py` then `vite build` |

## Aesthetic Direction: Editorial/Magazine

- **Typography:** Distinctive serif display font for headings (scholarly/journal feel), clean sans-serif for body and tables
- **Color palette:** Muted, warm — off-white background, dark charcoal text, one accent color (deep teal or burnt amber) for interactive elements and citation highlights
- **Layout:** Content-first, generous whitespace, light card shadows, thin dividers
- **Data emphasis:** Citation counts get visual weight (larger font, accent color). Track badges as small colored pills.
- **Dark mode:** Supported via CSS variables

## Paper Data Shape

```typescript
interface Paper {
  title: string;
  authors: string[];
  selection: string;        // "oral", "poster", "spotlight", "main", "findings", etc.
  keywords: string[];       // may be empty
  abstract?: string;        // present in enriched data or OpenReview
  link?: string;            // PDF link
  forum_id?: string;        // OpenReview forum ID
  citation_count?: number;  // present only in enriched data
}
```

## Conference Metadata (manifest.json)

```typescript
interface ConferenceMeta {
  id: string;               // e.g. "iclr_2025"
  venue: string;            // e.g. "ICLR"
  year: number;             // e.g. 2025
  paper_count: number;
  has_citations: boolean;   // whether enriched data exists
  tracks: string[];         // unique selection values
  top_papers: {             // top 3 by citation count (if enriched)
    title: string;
    citation_count: number;
  }[];
}
```
